1. Introduction.

scam n. 1 [1960s+] a plan, a scheme. 2 a large-scale plan to smuggle and distribute illegal drugs. 3 (US) information (cf. LOW-DOWN).

[GRE 2000]

Scam is an interpreter for a toy functional programming language that resembles a subset of Scheme [SPE 2007]. Scam programs are written in a parenthesized prefix notation. They can perform computations with lists, names (like Scheme’s symbols), numbers (32-bit two’s complement integers) and Unicode strings. Variables are lexically scoped, and procedures are properly tail recursive. Scam runs under Unix-like systems, such as GNU/Linux.
      Scam was written in the experimental programming language Orson [MOE 2014] as a test of the Orson compiler and parts of the Orson library. Since Scam is only a test, it omits many features that would be present in a complete programming language. However, it has been used by the author to implement a few simple applications, such as a program that formats parts of web pages.
      This manual gives a brief overview of Scam. It is not a tutorial. The reader is expected to be familiar with Scheme, or with another dialect of Lisp, such as Common Lisp [GLS 1990].

2. Characters.

Scam reads and writes text using a 31-bit character set with the UTF-8 encoding. This includes the familiar 7-bit ASCII character set and the 21-bit Unicode character set [UNI 2006] as subsets. It is possible to use Scam with ASCII characters on systems that do not support Unicode.
      Certain characters are used by Scam as delimiters and punctuation. They can be used as ordinary characters by prefixing them with backslashes. For example, a double quote can be used as an ordinary character by writing ‘\"’. A backslash itself can be used as an ordinary character by writing ‘\\’.
      Several common invisible characters may appear as letters prefixed by backslashes, as in the programming language C [KNR 1988]. Backslashed letters and the characters they represent are shown in the following table.

 LETTER 

 CHARACTER 

 LETTER

 Line feed (U+000A)

\B or \b

 Backspace (U+0008)

 Horizontal tab (U+0009)

\V or \v

 Vertical tab (U+000B)

Any character may be written as ‘\#’ followed by a series of one or more digits ‘0’ through ‘9’ and Roman letters ‘A’ through ‘F’, ‘a’ through ‘f’. The digits and letters denote the code for the character in hexadecimal, with digits having the values 0 through 9, and letters having the values 10 through 15. Codes range from 0 to 7FFFFFFF. For example, the string "λ-calculus" may also be written as "\#03BB-calculus", because the code for the character ‘λ’ is 03BB.
      Any numeric character code may be optionally followed by ‘\_’, which is ignored. This is sometimes necessary to separate a character code from the rest of a name or a string. For example, the string "\#03BB" has one character ‘λ’, but the string "\#03\_BB" has three: the first has the code 03, the second is ‘B’, and the third is also ‘B’.

3. Objects.

Scam supports seven kinds of objects, called conses, eofs, hooks, integers, lambdas, names, and strings. An eighth kind, called lists, are constructed from conses. All objects can be the bindings of names, can be passed as arguments to procedures, and can be written to text files. All objects except circular lists, eofs, hooks, and lambdas can be read from text files.
      Whitespace (blanks and newlines) is ignored in the written representations of objects, except where it is required as a separator. Comments are treated as whitespace: each comment begins with a semicolon ‘;’ and ends with a newline. Control characters (such as tabs) are also treated as whitespace, except for a few that the operating system uses as signals.

3.1. Conses.

A cons is a pair of objects, called (for historical reasons) the car and the cdr. It is written as (a . d), where a is the written representation of the car, and d is the written representation of the cdr.

3.2. Eofs.

An eof denotes the end of a text file. A procedure that reads objects from a text file will return an eof when there are no objects left to be read. Scam writes an eof as [eof], but this notation will cause an error if it is read back in again.

3.3. Hooks.

A hook is a procedure provided by Scam. Scam writes a hook as [h a], where h is a series of letters that identify the hook, and a is the hexadecimal memory address where the hook resides. This notation will cause an error if it is read back in again.

3.4. Integers.

An integer is a two’s complement integer between –2147483648 and 2147483647, inclusive. It is written as a series of one or more decimal digits ‘0’ through ‘9’. The first digit in the series may be optionally prefixed by a dash ‘–’, indicating a negative number.

3.5. Lambdas.

A lambda is a procedure provided by a user program. Scam writes a lambda as [closure a], where a is the hexadecimal memory address where the lambda resides. This notation will cause an error if it is read back in again.

3.6. Lists.

A list is an ordered series of zero or more objects, called its elements. It is written like this, where the subscripted o’s are written representations of the elements, separated by whitespace.

(o1 o2 ... on)

The list is an abbreviation for a series of nested conses:

(o1 . (o2 ... . (on . nil) ... ))

Such a list, whose rightmost cdr is the name nil, is called a proper list. An improper list is one whose rightmost cdr is an object other than nil. A proper list of zero elements may be written as (), and is equivalent to nil.
      Circular lists may be created by the procedures setCar and setCdr. When Scam writes a list, it will detect these circularities, and write each one as [...]. This notation for circularities will cause an error if it is read back in again.

3.7. Names.

A name is written as a series of one or more characters, other than brackets ‘[’ and ‘]’, dots ‘.’, double quotes ‘"’, parentheses, and whitespace. These and other characters may appear in names by prefixing them with backslashes.
      If two names are made up of the same characters, in the same order, then they refer to the same object in memory. Names may be therefore tested for equality by the procedure objEq, which compares the addresses of two objects. This does not necessarily work for objects other than names.

3.7.1. Private names.

If a name begins with a colon ‘:’, then Scam automatically prefixes it by the name of the text file being read when the name is created. For example, if Scam is reading the file /home/jim/scam/sort.scam, then the name :partition will be treated as if it was written sort:partition. If a name that begins with a colon is read from the standard input device, such as a keyboard, then the name is unchanged. This convention allows each Scam file to have its own private names that are distinct from the names in other files. Note that this is not how colons work in Common Lisp [GLS 1990].

3.7.2. Bindings.

Each name is optionally associated with an object, called the name’s binding. If a name has a binding, then it is said to be bound, otherwise it is said to be unbound.
      The act of associating a name with an object is called binding the name to the object. Most (but not all) names are initially unbound when Scam begins running. A name may be bound to many different objects as a Scam program runs, and may alternate between being bound and unbound.

3.7.3. Booleans.

The name nil is initially bound to itself. It represents both a proper list of zero elements, and a Boolean false value. Any object other than nil represents a Boolean true value. The name true is also initially bound to itself. It is often convenient to use true as a Boolean true value, even though any object other than nil would suffice.

3.8. Strings.

A string starts with a double quote ‘"’, is followed by zero or more characters other than newlines and double quotes, and ends with another double quote. The empty string, with zero characters, is written as two adjacent double quotes, ‘""’. Double quotes themselves, and any other characters, may appear in strings by prefixing them with backslashes.

4. Procedures.

A procedure is either a hook or a lambda. It may be applied to zero or more objects, called its arguments. An application of a procedure p to its arguments, the subscripted a’s, is written as a proper list:

(p a1 a2 ... an)

The application either returns exactly one object, called its value, or else returns no value at all. An application does not return a value if it is in error, or if it applies throw.

4.1. Errors.

Applying a procedure with the wrong types of arguments, with the wrong number of arguments, or both, will cause an error. For most errors, Scam applies throw, which terminates the most recently executing application of catch. If no application of catch is executing when the error occurs, then a message will be written to standard output, and control will pass back to Scam.

4.2. Tests.

If a procedure is said to test for something, then it will return either nil or true, depending on whether its arguments have the property being tested for. By convention, the names of most such procedures begin with is. For example, isCons tests if its argument is a cons.

4.3. Primitives.

Scam provides many primitive procedures. Each primitive procedure is implemented as a hook. Names are bound to these hooks so they may be used in programs. These names may be rebound to other objects.
      The next few sections describe all of Scam’s primitive procedures. The title of each section shows an application of the primitive procedure that the section describes. The names shown as the application’s arguments describe their expected types, as in the following table.

 ARGUMENT  

 EXPECTED TYPE  

b or o

 Any object.

c

 A cons.

k

 An integer.

n

 A name.

p

 A procedure.

s

 A string.

For example, the section titled (car c) describes a primitive procedure to which the name car is bound. This procedure expects one argument c, which must be a cons.

4.3.1. (and o1 o2 ... on)

Evaluate the arguments in order of their appearance. If one of the arguments returns nil, then return nil immediately without evaluating the rest. Otherwise return the value of the last argument. The application (and) returns true.

4.3.2. (car c)

Return the car of the cons c.

4.3.3. (caxr c)

Equivalent to (car (cxr c)), where x is a series of one or more a’s and d’s. See car and cdr.

4.3.4. (catch o1 o2 ... on)

If none of the arguments applies throw, then evaluate the arguments in order of their appearance, and return the value of the last argument, as if they appeared in last. If one of the arguments evaluates (throw b) while it is not evaluating another application of catch, then terminate the application of catch immediately, and return the value of b.

4.3.5. (cdr c)

Return the cdr of the cons c.

4.3.6. (cdxr c).

Equivalent to (cdr (cxr c)), where x is a series of one or more a’s and d’s. See car and cdr.

4.3.7. (cons o1 o2)

Return a cons whose car is the value of the first argument, and whose cdr is the value of the second. See car and cdr.

4.3.8. (cr o)

Return o.

4.3.9. (define n o)

Equivalent to (set n o). See set.

4.3.10. (first o1 o2 ... on)

Evaluate the arguments in order of their appearance, and return the value of the first argument. The application (first) returns nil.

4.3.11. (if o1 o2 o3)

Evaluate the first argument. If it returns nil, then evaluate the third argument and return its value. If it returns an object other than nil, then evaluate the second argument and return its value instead. The application (if b o) is equivalent to (if b o nil).

4.3.12. (imply o1 o2 ... on)

Evaluate the arguments, except the last, in order of appearance. If one of the arguments returns nil, then return true immediately, without evaluating the rest. Otherwise evaluate the last argument and return its value.

4.3.13. (include s)

If s has already appeared in a call to include, then do nothing. Otherwise load the file whose pathname is s. In either case, return s.

4.3.14. (intAdd k1 k2 ... kn)

Add the integer arguments in order of their appearance, and return the resulting integer. The application (intAdd) returns 0, and the application (intAdd k) returns the value of k.

4.3.15. (intAnd k1 k2 ... kn)

Compute the bitwise logical AND of the integer arguments, and return the resulting integer. The application (intAnd) returns –1, and the application (intAnd k) returns the value of k.

4.3.16. (intDiv k1 k2)

Divide the first integer by the second, and return the resulting integer. It is an error if the second integer is 0.

4.3.17. (intEq k1 k2)

Test if the two integers are equal.

4.3.18. (intGe k1 k2)

Test if the first integer is greater than or equal to the second.

4.3.19. (intGt k1 k2)

Test if the first integer is greater than the second.

4.3.20. (intLe k1 k2)

Test if the first integer is less than or equal to the second.

4.3.21. (intLt k1 k2)

Test if the first integer is less than the second.

4.3.22. (intMod k1 k2)

Return the remainder after dividing the first integer by the second. It is an error if the second integer is 0.

4.3.23. (intMul k1 k2 ... kn)

Multiply the integer arguments in order of their appearance, and return the resulting integer. The application (intMul) returns 1, and the application (intMul k) returns the value of k.

4.3.24. (intNe k1 k2)

Test if two integers are not equal.

4.3.25. (intNeg k)

Return the integer k with the opposite of its original sign.

4.3.26. (intNot k)

Return the bitwise logical NOT of the integer k.

4.3.27. (intOr k1 k2 ... kn)

Compute the bitwise logical OR of the integer arguments, and return the resulting integer. The application (intOr) returns 0, and the application (intOr k) returns the value of k.

4.3.28. (intSub k1 k2)

Subtract the first integer from the second and return the resulting integer.

4.3.29. (intXor k1 k2 ... kn)

Compute the bitwise logical exclusive OR of the integer arguments, and return the resulting integer. The application (intXor) returns –1, and the application (intXor k) returns the value of k.

4.3.30. (isCons o)

Test if o is a cons.

4.3.31. (isHook o)

Test if o is a hook.

4.3.32. (isEof o)

Test if o is an eof.

4.3.33. (isInt o)

Test if o is an integer.

4.3.34. (isLambda o)

Test if o is a lambda.

4.3.35. (isName o)

Test if o is a name.

4.3.36. (isNil o)

Test if o is nil. See not.

4.3.37. (isString o)

Test if o is a string.

4.3.38. (isλ o)

Test if o is a lambda. Equivalent to (isLambda o).

4.3.39. (lambda (n1 n2 ... nm) b1 b2 ... bn)

Return a new lambda whose parameters are the subscripted n’s. When the lambda is applied, its parameters are bound to their corresponding arguments. The subscripted b’s are then evaluated in order of appearance, as if they appeared in an application to last. The application then restores the parameters to their original bindings (or lack of bindings) and returns the value of the last subscripted b. Also see λ.
      There must be at least one subscripted b, but there may be zero subscripted n’s, so that (lambda () b) returns a lambda with no parameters. It is an error if any subscripted n appears more than once as a parameter in a single call to lambda.

4.3.40. (last o1 o2 ... on)

Evaluate the arguments in order of their appearance, and return the value of the last one. The application (last) returns nil, and the application (last o) returns the value of o.

4.3.41. (let ((n1 o1) (n2 o2) ... (nm om)) b1 b2 ... bn)

Simultaneously bind each subscripted n to the value of its corresponding subscripted o. Then evaluate the subscripted b’s in order of their appearance, as if they appeared in an application of last. Finally, restore the parameters to their original bindings (or lack of bindings) and return the value of the last subscripted b.

4.3.42. (list o1 o2 ... on)

Evaluate the arguments in order of their appearance and return a list whose elements are the values of these arguments. The application (list) returns nil.

4.3.43. (load s)

Open a file whose pathname is s. The file must have the suffix .scam. Read a Scam program from this file and evaluate it, then close the file and return s. It is an error if the file cannot be opened, if the Scam program read from the file has errors, or if the file cannot be closed.

4.3.44. (nameStr n)

Return a string whose characters are those of the name n.

4.3.45. (not o)

Test if o is nil. See isNil.

4.3.46. (objEq o1 o2)

Test if two objects reside at the same memory address: that is, test if they are identical objects.

4.3.47. (objLen o)

If the object o is a list, then return the number of elements in the list, an integer. The number of elements in the empty list nil or () is 0. It is an error if the list appears to have an infinite number of elements because it is circular. If o is not a list, then return 0.

4.3.48. (objNe o1 o2)

Test if two objects reside at different memory addresses: that is, test if they are not identical objects.

4.3.49. (objWrite o)

Write the object o to the standard output device, in such a way that it may be read back in again.

4.3.50. (or o1 o2 ... on)

Evaluate the arguments in order of their appearance. If one of the arguments returns a value other than nil, then return the value of that argument immediately, without evaluating the rest. Otherwise return nil. The application (or) returns nil.

4.3.51. (quit k)

Halt Scam and pass the value of the integer k back to the operating system. By convention, passing 0 to the operating system indicates that Scam has halted successfully, and an integer other than 0 indicates that it has halted with an error. The application (quit) is equivalent to (quit 0).

4.3.52. (quote o)

Return o without evaluating it. The application (quote o) may be abbreviated as ‘o (with an open quote) or as ’o (with a closing quote). Note that an open quote does not act the way it does in Common Lisp [GLS 1990.]

4.3.53. (set n o)

Bind the unevaluated name n to the value returned by evaluating the object o.

4.3.54. (setCar c o)

Change the car of the cons c to the object o, and return o. This may be used to create circular lists.

4.3.55. (setCdr c o)

Change the cdr of the cons s to the object o, and return o. This may be used to create circular lists.

4.3.56. (strAdd s1 s2 ... sn)

Concatenate the strings returned by the arguments into one string, and return the resulting string. The application (strAdd) returns the empty string "", and the application (strAdd s) returns the string s.

4.3.57. (strDate)

Return a string containing the day of the week, the date, the time, and the time zone. For example, "Monday, May 16, 2011 at 8:30 PM CDT".

4.3.58. (strEq s1 s2)

Test if two strings are lexicographically equal.

4.3.59. (strGe s1 s2)

Test if one string is lexicographically greater than or equal to another.

4.3.60. (strGt s1 s2)

Test if one string is lexicographically greater than another.

4.3.61. (strLe s1 s2)

Test if one string is lexicographically less than or equal to another.

4.3.62. (strLen s)

Return the integer number of characters in the string s.

4.3.63. (strLow s)

Return a string like s in which all upper case Roman letters (A–Z) are converted to lower case. Also see strUp.

4.3.64. (strLt s1 s2)

Test if one string is lexicographically less than another.

4.3.65. (strName s)

Return a name whose characters are those of the string s.

4.3.66. (strNe s1 s2)

Test if two strings are lexicographically unequal.

4.3.67. (strRead)

Read a line of text from the standard input device, and return it as a string. Return an eof if there are no more lines left to read.

4.3.68. (strSub s k1 k2)

Return a substring of the string s starting at the first integer index, and ending with the second integer index minus 1. (Common Lisp [GLS 1990] indexes also work this way.) It is an error if the indexes are out of range for the string s. The application (strSub s k) is equivalent to (strSub s k (strLen s)), except that s is evaluated only once. See strLen and strSub.

4.3.69. (strUp s)

Return a string like s in which all lower case Roman letters (a–z) are converted to upper case. Also see strLow.

4.3.70. (strWrite s)

Write the string s to the standard output device, without any punctuation such as double quotes ‘"’ or backslashes ‘\’. Then return s.

4.3.71. (throw o)

Return the value of o from the most recent application of catch that is currently executing. It is an error if no application of catch is being executed.

4.3.72. (λ (n1 n2 ... nm) b1 b2 ... bn)

Equivalent to (lambda (n1 n2 ... nm) b1 b2 ... bn). See lambda.

5. Programs.

A Scam program is typically a series of define’s that bind various names to lambdas. The names may be bound in any order, and the lambdas may call each other freely. For example, the following program implements a symbolic equation solver for middle school algebra. It can solve for any variable that appears exactly once in an equation. The equation may use the 2-ary operators ‘+’, ‘–’, ‘×’, and ‘/’.

;  Solve the equation leftSide = rightSide for variable.

(define solve 
 (λ (variable leftSide rightSide) 
  (if 
   (isInside variable leftSide) 
   (solving variable leftSide rightSide) 
   (solving variable rightSide leftSide))) )

;  Solve when variable appears in leftSide.

(define solving 
 (λ (variable leftSide rightSide) 
  (if 
   (objEq variable leftSide) 
   (list ‘= leftSide rightSide) 
   (if 
    (objEq (operator leftSide) ‘+) 
    (solvingAdd variable leftSide rightSide) 
    (if 
     (objEq (operator leftSide) ‘–) 
     (solvingSubtract variable leftSide rightSide) 
     (if 
      (objEq (operator leftSide) ‘×) 
      (solvingMultiply variable leftSide rightSide) 
      (if 
       (objEq (operator leftSide) ‘/) 
       (solvingDivide variable leftSide rightSide) 
       (throw "Unknown operator."))))))) )

;  Solve when variable is in leftSide, whose outer operator is +.

(define solvingAdd 
 (λ (variable leftSide rightSide) 
  (if 
   (isInside variable (left leftSide)) 
   (solving variable 
    (left leftSide) 
    (list ‘– rightSide (right leftSide))) 
   (solving variable 
    (right leftSide) 
    (list ‘– rightSide (left leftSide))))) )

;  Solve when variable is in leftSide, whose outer operator is –.

(define solvingSubtract 
 (λ (variable leftSide rightSide) 
  (if 
   (isInside variable (left leftSide)) 
   (solving variable 
    (left leftSide) 
    (list ‘+ (right leftSide) rightSide)) 
   (solving variable 
    (right leftSide) 
    (list ‘– (left leftSide) rightSide)))) )

;  Solve when variable is in leftSide, whose outer operator is ×.

(define solvingMultiply 
 (λ (variable leftSide rightSide) 
  (if 
   (isInside variable (left leftSide)) 
   (solving variable 
    (left leftSide) 
    (list ‘/ rightSide (right leftSide))) 
   (solving variable 
    (right leftSide) 
    (list ‘/ rightSide (left leftSide))))) )

;  Solve when variable is in leftSide, whose outer operator is /.

(define solvingDivide 
 (λ (variable leftSide rightSide) 
  (if 
   (isInside variable (left leftSide)) 
   (solving variable 
    (left leftSide) 
    (list ‘× (right leftSide) rightSide)) 
   (solving variable 
    (right leftSide) 
    (list ‘/ (left leftSide) rightSide)))) )

;  Test if variable appears inside expression.

(define isInside 
 (λ (variable expression) 
  (if 
   (isName expression) 
   (objEq variable expression) 
   (if 
    (isCons expression) 
    (or 
     (isInside variable (left expression)) 
     (isInside variable (right expression))) 
    false))) )

;  Return the outer operator of an expression.

(define operator car)

;  Return the left argument of an expression.

(define left cadr)

;  Return the right argument of an expression.

(define right caddr)

6. Running Scam.

When you invoke Scam from the shell, it responds with the prompt ‘Q:’. If you type a Scam expression at the prompt, then it will be evaluated, and the object it returns will be written to standard output, prefixed by ‘A:’. Note that Scam has no way to correct errors in multi-line inputs. If you make a typing mistake that can’t be fixed by backspacing, then all you can do is press control-C, which will throw away what you’ve typed, and get you another ‘Q:’ prompt.
      You can run the equation solving program shown above by applying load. Its argument is the pathname of the file that contains the program. The following shows how the program solves the equation y = mx + b for x. The solution is x = (y – b) / m.

Q: (load "scam/equation.scam") 
 
A: "scam/equation.scam" 
 
Q: (solve ‘x ‘y ‘(+ (× m x) b)) 
 
A: (= x (/ (– y b) m)) 
 
Q: (quit)

Scam continues in this way, asking for expressions and evaluating them, until you exit by typing control-D or applying quit.
